# 测试策略

## 哲学

写测试，但不要太多。大部分集成测试。

## 分层

| 层 | 占比 | 工具 | 测什么 |
|------|------|------|------|
| 单元 | 70% | Node: vitest / Python: pytest | 纯函数、工具类、规则引擎 |
| 集成 | 20% | supertest / pytest | API 端点、数据库操作 |
| E2E | 10% | Playwright / 手动 | 关键用户流程 |

## 覆盖目标

- 核心业务路径: 90%+
- 工具函数: 70%+
- UI 组件: 20%（E2E 覆盖关键流程）

## 质量门

PR 前必须过: lint → 单元测试 → 核心集成测试
